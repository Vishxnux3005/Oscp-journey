# 1. Networking Fundamentals

## Verifying Internet Connectivity

### Test Raw Connectivity

```bash
ping 8.8.8.8
```

Purpose:  
Checks whether the machine can reach the internet directly using an IP address.

Important:

- This bypasses DNS.
    
- If this works, internet routing/connectivity exists.
    

---

### Test DNS Resolution

```bash
ping google.com
```

Purpose:  
Checks whether DNS is functioning.

DNS translates:

```text
google.com → IP address
```

If:

- `8.8.8.8` works
    
- but `google.com` fails
    

then:

```text
Internet works
DNS is broken
```

---

# DNS Concepts

## What is DNS?

DNS (Domain Name System) converts human-readable names into IP addresses.

Example:

```text
google.com → 142.250.x.x
```

Without DNS:

- websites would only be accessible through IP addresses.
    

---

## DNS Configuration File

```bash
cat /etc/resolv.conf
```

Purpose:  
Displays DNS resolver configuration.

Typical output:

```text
nameserver 8.8.8.8
```

Meaning:  
The system will use that DNS server to resolve domain names.

---

# Important Networking Troubleshooting Logic

## Isolating Variables

Meaning:  
Troubleshoot one possible issue at a time.

Possible variables:

- DNS broken
    
- router issue
    
- VPN issue
    
- firewall issue
    
- network adapter issue
    
- no IP assigned
    

Example workflow:

1. Check internet using IP
    
2. Check DNS using domain name
    
3. Check interface/IP
    
4. Check VPN adapter
    
5. Check routes
    
6. Check firewall
    

This is called:

```text
Systematic troubleshooting
```

---

# VirtualBox Networking Issue Encountered

Problem:

```text
Network is unreachable
```

Observed:

```bash
ip a
```

showed:

```text
127.0.0.1
```

Meaning:  
Only loopback/self-networking existed.

The VM had no real network connectivity.

Fix:

- Enable VirtualBox network adapter
    
- Use NAT networking
    

After fix:

```text
10.0.2.15
```

was assigned.

Meaning:  
The VM successfully obtained a network IP.

# Bash Scripting Foundations

## What is Bash?

Bash is:

```text
A shell / command-line interpreter
```

It acts as the interface between:

- user
    
- operating system
    

Flow:

```text
User → Bash → Linux OS → Result
```

Purpose:  
Authentication mechanism replacing GitHub passwords.

Important:

- behaves like a password
    
- should never be shared publicly
    

Required permission:

```text

```
